🔧 XState v4 (Legacy)

  4. 복합 주문 시스템

⚡ XState v5 (Modern)

현대적 방식, Actor 모델 기반

  1. 기본 개념 및 상태 전환
  2. 기본 주문 시스템
  3. 결제 및 배송 처리
  4. 복합 주문 시스템
  5. ERP 판매주문 기본
  6. 재고 및 출고 처리
  7. 완전한 ERP 프로세스

💡 주요 차이점

  • v4: createMachine + transition
  • v5: createMachine + createActor
  • v5에서 비동기 처리가 더 안정적
  • v5는 fromPromise, fromCallback 등 명확한 API 제공
